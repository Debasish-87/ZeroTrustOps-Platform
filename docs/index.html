ZeroTrustOps Platform

Self-hosted security enforcement platform for cloud-native infrastructure
Go 1.21+ Python 3.11 Apache 2.0 Docker Compose PostgreSQL 16

Overview

ZeroTrustOps is a self-hosted DevSecOps platform that automatically scans repositories for security misconfigurations on every push. It integrates a custom-built static analysis engine (SecTL), secrets detection, and a real-time dashboard — deployable in a single command.

01 — Developer pushes code to GitHub
02 — GitHub fires a webhook to the platform API
03 — API clones the repository into an isolated workspace
04 — SecTL scans Kubernetes manifests, Terraform, and Helm charts
05 — Gitleaks scans for hardcoded secrets and credentials
06 — Findings stored in PostgreSQL, gate decision computed
07 — Dashboard displays PASS or FAIL with full finding detail

Architecture

CLI Engine
SecTL
Go
Custom static analysis engine. Scans K8s, Terraform, Helm, and container images. 70+ built-in rules. Outputs table, JSON, and SARIF.
Backend API
Platform API
Python / FastAPI
Receives GitHub webhooks, orchestrates background scans, stores results, and exposes a REST API for the dashboard.
:8000
Frontend
Dashboard
React + Vite + Nginx
Real-time scan results, repository management, finding detail view, and webhook setup guide.
:3000

All three services are orchestrated via Docker Compose and communicate over an internal bridge network. PostgreSQL stores all scan state persistently.

Output Formats

Usage
$sectl scan ./manifests --type k8s  # human-readable table
$sectl scan ./manifests --type k8s -o json  # machine-readable JSON
$sectl scan ./manifests --type k8s -o sarif  # GitHub Security tab
$sectl scan ./infra --type terraform --severity high --fail-on-findings
$sectl verify nginx:latest myapp:1.0.0  # supply chain
$sectl audit --provider aws --region us-east-1  # live posture

Quick Start

Prerequisites

Docker+ Compose
Go1.21+
Gitany
ngrokfor webhook tunnel

Installation

Shell
$git clone https://github.com/Debasish-87/ZeroTrustOps-Platform.git
$cd ZeroTrustOps-Platform
$bash setup.sh

The setup script checks prerequisites, compiles the SecTL binary from source, builds all Docker containers, and waits for health checks to pass.

ServiceURL
Dashboardhttp://localhost:3000
APIhttp://localhost:8000
API Reference (Swagger)http://localhost:8000/docs

GitHub Webhook Setup

Shell
$ngrok http 3000  # exposes local platform publicly

Add the generated URL as a webhook in your GitHub repository settings under Settings > Webhooks. Set the payload URL to https://your-ngrok-url/webhook/github with content type application/json and trigger on push events.

Uninstall

Shell
$bash uninstall.sh  # removes all containers, volumes, images, and sectl binary

Kyverno Policies

Three cluster-wide policies are included, all running in Enforce mode — they actively block non-compliant resources from being admitted to the cluster.

PolicyAction
disallow-latest-tagBlocks containers using the :latest image tag or no tag at all
disallow-privilegedBlocks privileged containers, privilege escalation, and host namespace sharing
require-resource-limitsRequires CPU and memory requests and limits on all containers
